# 吹き出しのルール

- 各セリフの吹き出しは話者の口元の近くに配置する
- テイル (尾の三角) は話者の方向を指す
- 2 人以上のキャラが同じパネルにいる場合、話者ではないキャラの方には吹き出しを置かない
- **吹き出しの中には「セリフ本文だけ」を入れる**。話者名は書かない
- 吹き出しは白いベタ塗り + 黒文字 (ダーク背景上で読みやすく)
- **話者名タグ**: 各パネルの左下隅に「ピンク背景 #FF2D7A + 白文字」の小さな角丸タグ
  (フレームの番号円とは別に、パネル内側の左下に追加配置)
- 1 パネルに複数の話者がいる場合はタグを左下にまとめて並べる

# 整合性ルール

- キャラの服装・配色・小道具は char リファレンス画像に厳密に従う
- ラボ内装は env リファレンス画像のレイアウトを尊重する
- アイリスは抽象 AI として扱う (特定のベンダー / モデル / サービスを連想させない)
- バグまるは口を描かず、セリフを与えない (効果音「ぴょこ」「ぴゃっ」のみ)
- モナカは犬語のみ (人語を喋らせない)
- 既存企業 / プロダクトのロゴ・名前を画面に出さない

## エピソード情報

- **エピソードタイトル**: 起動完了しました、たぶん
- **締めの一文**: [アイリスにも、朝はあるらしい。]

- シーズン: S1
- arc: standalone

## このエピソードのコア（読者に伝えるべき因果）

1. **コマ 1-2**: シュンタとモナカが入室 → コーヒーマシン起動 (アイリス起動の合図)
2. **コマ 3**: アイリスのホログラムは点くが **目が半目で焦点が合わず、寝起きの様子**
3. **コマ 4**: シュンタが声をかけて初めてアイリスがハッとして取り繕う
   → **「AI なのに寝起きが悪い」** という発見でオチる

→ アイリスのキャラ性（真顔のボケ / どこか抜けてる）を第一弾として確立する。
読者が「AI なのにこんな瞬間あるんだ」とクスッと笑える因果を 1 枚に収める。

## 登場キャラ

- シュンタ, モナカ, アイリス

- 話者: シュンタ

  - 起動ボタンが押された瞬間、シアンブルー #6EE7F7 のインジケーターランプが点灯

- 時刻: 朝

- 場面: シュンタのデスク中央、ホログラム位置にアイリスがふわっと立ち上がる
  - 起動エフェクトのパーティクルが **少なめ / 弱め** (いつもより勢いがない)

- 話者: アイリス

- 場面: 同じデスクの前
  - シュンタが少し前のめりで「アイリス…？」と声をかける

- ゾーン: 執務エリア
- 時刻: 朝

- 動作・表情: シュンタは「えっ」の顔、アイリスは取り繕い気味の早変わり、モナカはきょとん
- 話者: アイリス
- 吹き出し: あっ、おはようございます！ 起動完了しました！
- キャラ名タグ: アイリス（パネル左下、ピンク + 白）
- 効果音 / モノローグ:

  - モノローグ（シュンタ）「(……AI も、朝あるんだ)」 ← 発見オチを明示

## 描画上の念押し

- **コマ 3 の "半目アイリス" がオチの伏線** — ここで読者が「あれ？」と感じる必要がある。ディスプレイの表情を明確に「(- _ -)」と分かる形にする
- **コマ 4 の表情早変わり** は、コマ 3 とのコントラストで「ハッと取り繕った」感を出す。「(- _ -)」 → 「(^_^)」の変化を強調
- コマ 3 のセリフ「…………うぃ」は寝起きのうわごと感。長い沈黙 + 弱い音
- コマ 4 の「起動完了しました！」はやや早口・取り繕い感（!マーク多めでもいい）
- **AI のセリフルール厳守**: アイリスにベンダー名 / モデル名は言わせない
