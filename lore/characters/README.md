## キャラファイルの書き方ルール

- 1 ファイル 1 キャラ
- ファイル名はローマ字小文字（ID と一致させる）
- 先頭に front matter を置く（エピソード側で参照しやすくするため）
- **設定変更は必ず PR にする**（変更履歴を Git に残すのがこのリポジトリの肝）
- 新規キャラは [`./_template.md`](./_template.md) をコピーして書き始める

## ファイル構成（共通セクション）

各キャラファイルは以下のセクションを持つ:

| セクション         | 内容                                                          |
| ------------------ | ------------------------------------------------------------- |
| 概要               | 役割と性格の 1 段落要約                                       |
| 役割               | ラボでの担当                                                   |
| 強み・得意なこと   | 物語上で活かす能力                                            |
| 苦手・弱み         | コメディ / 葛藤の燃料                                          |
| 好きなもの         | 小道具 / 雑談ネタの起点                                       |
| 口調・口ぐせ       | セリフのトーン統一                                             |
| バックストーリー   | 過去・転機・なぜ今の性格になったか                              |
| 内面の葛藤         | 表に出していない不安・矛盾（描き手用）                          |
| 成長アーク         | S1 → S4 でどう変化するか                                       |
| 言ってはいけないライン (NG) | キャラ崩壊を防ぐガード                                |
| 仕草・小道具の癖   | 描き分けのフック                                                |
| ビジュアル         | 衣装 / 配色 / 表情パターン                                      |
| デジラボでの動き   | 普段の行動                                                     |
| デジラボに参画した経緯 | origin の要約                                                 |
| 関係性             | 他キャラとの距離感（詳細は relationships へ）                  |

## キャラ追加時のチェックリスト

- [ ] [`./_template.md`](./_template.md) をコピーして `characters/<id>.md` を作成
- [ ] front matter（id / name / role / tags 等）を埋める
- [ ] バックストーリー / 内面の葛藤 / 成長アーク を埋める
- [ ] 言ってはいけないライン (NG) を 3〜5 個書く
- [ ] このページの一覧表に追記
- [ ] [`../relationships.md`](../relationships.md) に関係性を追記
- [ ] [`../lexicon.md`](../lexicon.md) に名前 / 表記ルールを追記
- [ ] [`../timeline.md`](../timeline.md) に登場・参画タイミングを追記
- [ ] 立ち絵 / アイコンを [`../../assets/characters/<id>/`](../../assets/characters/) に追加
