# Season 1 — デジラボ、はじめる

> 主プロジェクト: **デジラボ公式サイト + ポートフォリオ**
> テーマ: **キャラ紹介・日常確立・シリーズの基盤**

## このシーズンの位置付け

シリーズの "基礎工事"。読者にとっての初めての出会いになる回が多いシーズン。
- メンバーがどういう人か、どこで何をしているかが伝わること
- "デジラボらしさ"（明るい / ものづくり中心 / AI が当たり前にいる）が体感できること
- 笑いの方向性（技術ネタ・デザインネタ・あるある）の地ならし
- バグまるの存在を匂わせ、長期伏線の種をまく

## 物語の起点

- Shunta が「ものづくりの裏側を漫画で残してみよう」と決める
- ついでにデジラボ公式サイトも作ろう、と話が広がる
- 「ラボのアウトプットを世に出す」という S1 全体のゆるい目標になる

## 主な出来事（このシーズンで描きたいハイライト）

1. **メンバーの紹介回** — Shunta / アイリス / ミカ / タクマ が日常の中で立つ回を 1 本ずつ
2. **デジラボサイト着手** — 雑談から「サイト作ろう」になる流れ
3. **デザインバトル** — サイトのデザインで Shunta vs ミカが揉める / タクマが裁定者
4. **バグまる初登場** — 中盤、本番環境にバグまる初登場
5. **倒しても増える** — バグまる、倒しても増える / でも憎めない描写を積む
6. **季節回** — 桜・梅雨・夏祭り・忘年会・正月を機会を見て挟む
7. **アイリスの不可解ログ** — 終盤、アイリスのログに不可解な記録 → S2 への引き

## "あるある" 在庫（S1 で使いやすいネタ）

- ドキュメント・README ネタ
- 命名・型・環境構築の沼
- AI に頼って逆に困る
- 動いた・何も触ってないのに
- フォント論争 / デザインレビュー
- コーヒーマシン故障
- リファクタリング誘惑に負ける
- 素数を選ぶエンジニア（外食回）
- 「あとでやる」が積まれていく

→ 詳細は [`../../../ideas/backlog.md`](../../../ideas/backlog.md) で管理

## キャラの S1 ステート

| キャラ   | S1 序盤                                  | S1 終盤                                              |
| -------- | ---------------------------------------- | ---------------------------------------------------- |
| Shunta   | "やってみよう" のテンションで突っ走る    | サイトリリース達成。次は "プロダクトを世に" を意識し始める |
| アイリス | 日常の相棒として淡々と機能する          | ログの不可解さを匂わせる（伏線）                      |
| ミカ     | デザイン担当として Shunta にダメ出し     | ラボのトーンを確立した自負を持ち始める                |
| タクマ   | 締切で現実に戻す調整役                   | "外に出す" の重要性を強調し始める（S2 への橋渡し）    |
| バグまる | 中盤で初登場、軽い小ネタとして          | "倒しても増える / 憎めない" マスコット位置を確立      |

## バグまるアークでの段階

[`../arcs/bugmaru.md`](../arcs/bugmaru.md) の段階で言うと：

- **萌芽**（中盤）: Shunta が一瞬 "何か" を見る。アイリスのカメラには映らない
- **接触**（後半）: バグまる、本番環境に出現。倒しても増える

## シーズン末の小ピーク

- デジラボサイトの公開（プロジェクト山場）
- 公開後のログにバグまる的シルエットが映り込む（伏線一段進める）
- アイリスの初期コミットログを Shunta が読み返したくなる、という S2 へのフック
